Исходя из изображения выше, создайте проект, в котором нужно разработать систему/иерархию классов, составляющих Кофемашину.
Используйте все известные концепции ООП для создания иерархии с учетом следующих рекомендаций:

Должен существовать основной класс с названием Кофемашина, который состоит из различных компонентов 
(в вашем усмотрении, какие компоненты создать и использовать) и предоставляет публичные методы (удобные для пользователя)
для взаимодействия со всей системой (например, вынуть чашку, добавить чашку, добавить воду и т.д.). Ваш подход к написанию
компонентов/системы должен быть как можно ближе к естественному языку, структура системы должна быть максимально приближена
к физической кофемашине. В точке входа в приложение необходимо создать экземпляр Кофемашины и предоставить способ взаимодействия
с пользователем через консоль (опции должны быть представлены в цикле, который завершается, когда пользователь больше не желает
взаимодействовать с Кофемашиной).
